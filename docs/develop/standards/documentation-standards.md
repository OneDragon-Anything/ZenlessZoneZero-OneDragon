# 文档规范

本文档规定了项目文档编写需要遵守的规范。

## 核心原则

### 方案优先

文档的核心目标是**将方案描述清晰**，而非提供完整的代码示例。读者应该通过理解方案来指导开发，而不是复制粘贴代码。

### 简洁性

- 避免冗长和重复的说明
- 只包含必要的信息，去除无关内容
- 使用简洁的语言表达

### 避免代码示例

- **禁止完整的代码实现**：文档不应包含可直接复制使用的完整代码
- **避免使用示例**：优先使用文字描述、流程图、架构图等方式说明
- **例外情况**：仅在以下情况允许极简代码片段
  - 说明配置格式（如 YAML、JSON 结构）
  - 说明 API 签名（必需参数和返回值）
  - 说明数据模型关键字段
