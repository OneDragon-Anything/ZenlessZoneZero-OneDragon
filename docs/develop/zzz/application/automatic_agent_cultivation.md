# 代理人自动培养

功能通过整合游戏系统源数据和用户仓库数据，实现代理人的自动化培养和培养路线规划。


## 1.核心玩法

通过以下核心步骤实现代理人的完整培养流程：
- 数据准备：从游戏系统获取代理人基础信息、技能升级需求、驱动盘配置等
- 仓库扫描：扫描用户当前拥有的材料、驱动盘、音擎等资源
- 培养规划：根据用户目标和当前资源，规划最优培养路径
- 自动执行：调用各模块执行代理人强化、驱动盘强化、音擎强化等操作


## 2.脚本流程

1. 初始化系统环境，加载游戏数据
2. 调用信息管理模块获取代理人基础配置
3. 扫描用户仓库，获取当前拥有的资源信息
4. 根据用户配置的培养目标，生成培养计划
5. 执行培养计划，依次调用：
   - 代理人自动强化
   - 驱动盘自动强化
   - 音擎自动强化
6. 完成后返回执行报告


## 3.支持配置

- 目标代理人选择 - 选择要培养的目标代理人
- 培养优先级设置 - 设定强化、驱动盘、音擎的优先级
- 目标等级设定 - 设定代理人目标等级、技能等级等
- 资源限制配置 - 限制消耗的材料数量、丁尼上限等
- 培养方案选择 - 预设的培养方案（全力培养、稳步培养、资源有限培养）


## 4.迭代方向

- 增加自动战斗获取材料的功能
- 支持多个代理人的批量培养计划
- 提供更细粒度的驱动盘筛选和配置
- 优化培养路径算法，平衡资源消耗和培养效率

## 仓库扫描

此模块为代理人自动培养的子模块之一，负责扫描用户当前拥有的代理人、驱动盘、音擎等资源信息。

### 1. 核心玩法

仓库扫描通过 OCR 识别技术自动采集游戏中的资源数据：

- **预扫描**：遍历所有已解锁代理人，记录代理人名称和代码
- **特定扫描**：对特定代理人进行详细扫描，获取驱动盘、音擎等装备信息
- **并行处理**：OCR 识别与界面操作并行执行，提高扫描效率

### 2. 脚本流程

```
开始仓库扫描 → 返回大世界 → 准备截图文件夹 → [预扫描/特定扫描]
```

**预扫描流程**（更新代理人列表）：
```
导航到代理人信息界面 → 遍历代理人列表 → OCR识别名称 → 模糊匹配翻译 → 写入agent_names.json
```

**特定扫描流程**（扫描特定代理人装备）：
```
导航到代理人信息界面 → 定位特定代理人 → 扫描代理人基础信息 → 扫描驱动盘 → 扫描音擎 → 写入数据文件
```

### 3. 数据来源

| 数据类型 | 来源文件 | 说明 |
|---------|---------|------|
| 角色翻译 | `assets/game_data/agent/_od_merged.yml` | 角色中文名与代码的映射 |
| 驱动盘翻译 | `assets/game_data/drive_disk/_od_merged.yml` | 驱动盘名称与代码的映射 |
| 音擎翻译 | `assets/game_data/engine_weapon/_od_merged.yml` | 音擎名称与代码的映射 |
| 画面定义 | `assets/game_data/screen_info/_od_merged.yml` | 界面区域和导航路径 |
| 扫描结果 | `.debug/inventory_data/` | 输出的 JSON 数据文件 |

### 4. 支持配置

- **扫描选项** - 更新代理人列表 / 选择特定代理人
- **代理人列表** - 从 agent_names.json 加载已扫描的代理人

### 5. 关键组件

| 组件 | 职责 |

| `AgentParser` | 代理人详细信息解析器 |

### 6. 解析器架构

所有解析器继承 `BaseParser` 抽象基类，提供统一的接口规范：

```python
class BaseParser(ABC):
    @abstractmethod
    def parse(self, ocr_result, *args, **kwargs) -> Optional[dict]: ...

    @abstractmethod
    def get_supported_type(self) -> str: ...

    def preprocess(self, text: str) -> str: ...
    def postprocess(self, result: dict) -> dict: ...
```

**解析器类型映射**：

| 解析器 | 类型标识 | 说明 |
|--------|----------|------|
| `AgentParser` | `agent` | 代理人详细信息解析 |
| `DriveDiskParser` | `drive_disk` | 驱动盘属性解析 |
| `WengineParser` | `wengine` | 音擎属性解析 |

## 一键调优

此模块为代理人自动培养的子模块之一，负责从用户仓库中自动查找并装备当前配置下评分最高的驱动盘。

### 1. 核心玩法

一键调优通过以下步骤实现驱动盘的自动优化：

- **导航到代理人**：自动导航到目标代理人的装备详情界面
- **套装筛选**：通过游戏内UI筛选出目标驱动盘套装
- **仓库捕获**：逐行扫描屏幕上显示的驱动盘，点击后通过OCR识别详细属性
- **评分排序**：根据代理人权重配置，计算每个驱动盘的相对评分并排序
- **装备最优**：自动滚动到评分最高的驱动盘位置并点击装备

### 2. 脚本流程

```
开始一键调优
  ↓
回到大世界
  ↓
导航到代理人-信息界面 → 遍历代理人列表OCR识别 → 定位目标代理人
  ↓
导航到代理人-装备详细界面
  ↓
执行驱动盘套装筛选
  ├─ 点击"对比" → 点击"筛选"
  ├─ 确保筛选条件为"获得顺序"
  ├─ 点击"重置" → 选择"驱动盘状态-其他" → 选择"S"品级
  ├─ 点击"套装选择" → 在套装列表中查找目标套装
  └─ 确认筛选
  ↓
获取所有驱动盘信息并保存
  ├─ _capture_initial_drive_disks: 捕获初始屏幕驱动盘
  │   ├─ _detect_drive_disk_grids: 检测驱动盘方格
  │   ├─ _capture_drive_disks_from_rows: 逐行点击并提交OCR
  │   └─ _detect_progress_bar: 检测是否触底
  ├─ _capture_scrolled_drive_disks: 未触底时滚动继续捕获
  ├─ _wait_and_get_ocr_results: 等待OCR完成，过滤非目标套装，计算评分
  ├─ _find_and_click_best_drive_disk: 排序找最高分，定位并点击
  │   ├─ _scroll_to_target_position: 滚动到目标行列
  │   ├─ _click_and_verify_drive_disk: 点击并OCR验证
  │   └─ _verify_drive_disk_info: 比对预期与实际信息
  └─ _save_drive_disk_data: 保存数据到JSON文件
```

### 3. 过滤机制

在 `_wait_and_get_ocr_results` 中通过套装过滤排除非目标驱动盘：

- OCR识别出每个驱动盘的套装代码（`setKey`）
- 与用户选择的目标套装（`selected_drive_disk_set`）进行比对
- 不一致的驱动盘直接跳过，不计入有效列表
- 过滤后重新生成连续ID（`zzz_disc_1`, `zzz_disc_2`, ...）
- 统计过滤数量并记录日志

### 4. 评分机制

使用 `InventoryDataProcessor.calculate_disc_score_formatted()` 计算驱动盘评分：

- 将OCR识别的英文属性键（如 `crit_`）通过 `SLOT_MAPPING` 转换为中文（如 `暴击率`）
- 从代理人YAML配置中读取中文权重值（如 `character_weight['暴击率'] = 0.75`）
- 计算主词条得分（仅4-6号位）和副词条得分
- 计算该位置的理论最大得分作为归一化分母
- 返回相对评分 `relativeScore`（0-100），跨位置公平可比

### 5. 位置定位

驱动盘按"获得顺序"排列在4列网格中，通过ID计算行列位置：

```
ID → 行列公式：row = (id-1)//4 + 1, col = (id-1)%4 + 1
```

滚动定位策略：计算目标行与当前窗口的位置差，自动滚动到目标附近。
