# 战斗系统建档

> 本文记战斗画面的识别机制 + 玩法要点 + 自动化关键约束。
> 截图来源:用户提供(.debug/images/_1784380*.png,2026-07-18)。

## 画面识别

### 战斗画面(实战动作界面)
- **screen_name**: `战斗画面`
- **识别方式**: `按键-普通攻击` 模板匹配(conf ~0.97)。**这是唯一稳定标识**——其余 UI 元素都随场景动态变化:
  - 右上角:精英怪 → **怪物血条**(boss HP);限时关卡 → **倒计时**。普通战斗两者都没有。
  - 血条/计时/连携条/技能图标/角色头像 → 随角色、敌人、数值变化,不可做标识。
  - battle.yml 中对这些动态元素有 area(`式舆防卫战-倒计时`/`式舆防卫战-倒计时-精英`),但仅作检测用(非 id_mark)。
- **id_mark 状态**: battle.yml 中**无任何 area 设 `id_mark: true`** → `is_precise=false`(非精确匹配)。
- **为什么可接受**:框架不靠 screen_name 精确匹配判战斗态,而是直接调 `is_normal_attack_btn_available()`(攻击按钮模板在不在)→ 返回 `in_battle` bool。所以"战斗画面"的识别不依赖 screen 图匹配,依赖攻击按钮存在性。
- **建档建议**: 若要让 screen_name 匹配也精确,可给 `按键-普通攻击` 设 `id_mark: true`(当前不设也能工作,因为框架走攻击按钮直查)。
- **进入战斗-菜单**:战斗中按 esc 或点左上角 → 弹出战斗-菜单(暂停)。

### 战斗-菜单(暂停菜单)
- **screen_name**: `战斗-菜单`(battle_menu.yml)
- **识别方式**: `菜单-返回` 模板(top-left 返回箭头) + `按钮-设置`/`按钮-战况` 文本 → **is_precise=true**(精确匹配)。
- **关键 area**: 按钮-退出战斗、按钮-重新开始、按钮-战况、按钮-设置。

### 战斗结果
- **screen_name**: `战斗画面`(同一 screen_name,不同 area status)
- **关键 area**: 战斗结果-完成、战斗结果-再来一次、战斗结果-撤退、战斗结果-已达成、战斗结果-退出、战斗结果-倒带。
- 识别靠这些 area 的文本/模板匹配(在战斗画面 screen 下按 area 区分状态)。

### 战斗-菜单 more(退出战斗子菜单)
- `battle_menu.yml` 含 按钮-退出战斗 → `ExitInBattle` operation 使用(点退出→等确认框→点确认→等消失)。

## 玩法要点(玩家视角)

- **战斗目标**: 击败敌人(普通副本)或达成条件(专业挑战:限时/无伤等)。
- **核心操作**: 普通攻击/特殊攻击/终结技/连携技/闪避/换人。
- **战斗结束条件**: 敌人全灭(成功)/角色全灭(失败)/主动撤退。
- **战斗中可暂停**: 按 esc → 战斗-菜单(可看战况/设置/退出/重新开始)。

## 自动化关键约束(给 PR 审查/开发)

- **战斗态判定**: `is_normal_attack_btn_available(screen)` → 攻击按钮模板在不在 → `in_battle`。**不依赖 screen_name 精确匹配**。
- **自动战斗停止**: `AutoBattleApp.after_operation_done` 调 `stop_auto_battle()`(多步:stop_running + stop_context → _release_keys/switch_*)。需 try/finally 保证基类清理必跑(#2459)。
- **战斗结束检测**: `check_battle_state` 里 `_check_battle_end` 不受 `in_battle` 限制(反缩进修复 #2372)——避免攻击按钮误匹配结算画面导致永不检测结束。
- **退出战斗**: `ExitInBattle` 用 `until_find_all`/`until_not_find_all` 做闭环(点退出→等确认→点确认→等消失),被 BackToNormalWorld/NotoriousHunt/AreaPatrol/CombatSimulation 等复用。
