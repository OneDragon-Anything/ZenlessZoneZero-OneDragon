# 随便观(经营玩法)

澄辉坪「随便观」经营玩法:周期托管 + 多子玩法(游历 / 饮茶仙 / 制造坊 / 售卖铺 / 好物铺 / 邦巢 / 德丰大押)。**重 app** —— 主 app `SuibianTempleApp` 调度 7+ 子 op,每子 op「从入口进 → 操作 → 返回入口」。本文基于**实拍 22 张 webp** + screen_info + 代码三层。

## 入口 + 状态流转(实拍确认)

大世界 → `Transport(澄辉坪, 随便观)` → **interact(F) 狮耶**(传送点旁,`>`狮耶`<` 可交互态)→ 狮耶对话(「振兴随便观」)→ click 对话框推进 → **随便观-入口**。

> ✅ 入口是 **interact 狮耶**(传送点旁 NPC,`>`名字`<` 可交互),**不是** OCR「前往随便观」文字。app `goto_suibian_temple` 的 `round_by_ocr_and_click_by_priority(['前往随便观',...])` 点的是 interact 狮耶后屏幕出现的交互提示文字。Transport 后传送点就在狮耶交互范围内(**app 无 move,不走**;走了反而过头错过交互点)。

入口两态:
- **自动托管中态**(入口-自动托管中.webp):中央「LV.45 / 完成复兴 / 自动托管中 / 可关闭自动托管进行手动操作」;游历/经营 tab **隐藏**(锁)。
- **手动态**(入口-手动态.webp):托管停止后,「游历」+「可派遣」/「经营」+「可制造」tab 显示 + 「暂停营业」。

**自动托管中 → 停止托管路径**(实拍):入口左下「托管中」(`129,967`,左半屏)→ 自动托管/经营日志画面 → 「停止托管」(`1159,879`)→ 确认弹窗(警告不足时间无收益)→ 获得奖励(百通宝/云纹徽结算)→ 入口手动态。

app 流程分两条(`config.auto_manage_enabled`,托管与子玩法**互斥** —— 托管中入口 tab 锁,没法手动派遣/制造):
- **开(默认)**:识别初始 → (传送 → interact 狮耶) → `handle_auto_manage`(`SuibianTempleAutoManage`:停止托管领收益 → **click「开始托管」重启托管**)→ 返回大世界。**不跑子玩法**(`handle_adventure_squad` 仅 `@node_from(..., status='未开启自动托管')`)—— 托管跑收益。
- **关**:识别初始 → (传送 → interact 狮耶) → `handle_auto_manage`(跳过,返 '未开启自动托管')→ 游历 → 饮茶仙(条件) → 饮茶仙后再游历 → 制造坊 → 售卖铺 → 好物铺(条件) → 邦巢(条件) → 德丰大押(代码未开启) → 返回大世界(手动子玩法)。

- **id_mark**:`按钮-游历小队`(`1540,30-1730,75`,显示「游历小队:X/4」X=可收获数)/ `菜单-返回`/ `按钮-街区`(`234,20-394,80`)。

**副本选择(踩坑)**:副本**名字不可点**,要点名字**上方 offset(0,-100)**选中(参考 `choose_mission` 代码 `offset=Point(0,-100)` —— 点 OCR 名字 center 上方 100px = 副本图标位);**选中态 = 名字左右 `>` `<` 三角形**(OCR 常漏三角符号,要 vision 看);切副本后右侧详情跟着切,但**副本详情文本结构相同**(SCORE / 物资 / 奖励都一样),OCR/vision 都看不出区别 —— 只能靠 `>`名字`<` 标记确认选中谁。

**副本态(⚠️ 易误判,实拍区分)**:
- **选时态(未派遣)**:右侧「剩余XX时XX分」(**是选时间的预览,非进行中倒计时!**)+「请选择邦布」+「自动选择邦布」+「派遣」按钮。
- **已派遣态**:右侧「剩余XX时XX分」+ **「游历中」标签** + **「提前收获」按钮**(`choose_period` 检测「提前收获」→ `already_dispatch`,跳过选时)。
- 判据:**有「游历中」标签 +「提前收获」按钮才是真已派遣**;vision 看「剩余时间」猜「已派遣」是错的(选时态也有「剩余时间」预览)。

**派遣流程(实拍)**:选副本 → 选子副本(`choose_sub_mission` 点 `标题-子副本-N`,位置待实拍确认)→ `AdventureDispatch`:选时间(点 `按钮-选择时间` → `弹窗-游历时间选择` 选 duration)+ 自动选择邦布(`按钮-自动选择邦布`)+ 派遣(`click_dispatch` OCR「派遣」)。
- **选时间弹窗**:6 选项(15分钟 / 1 / 2 / 6 / 12 / 20 小时),制造区**无 3 分钟**(最短 15 分钟;3 分钟可能在非深处副本,待确认)。
- ⚠️ **派遣无确认弹窗**:点「派遣」直接成功(走 `dispatch_success` 无弹窗分支,`click_dispatch_confirm_dialog` 常规不触发)。派遣后游历小队 X/4 +1、副本转「游历中」态。

**收获流程(待实拍)**:游历小队到期完成(可收获 X>0)→ 点游历小队 → `click_finish`(游历完成)→ `click_claim`(可收获)→ `click_confirm`(确认)→ 自动选择邦布。收获链 fixture 待拍;「提前收获」(已派遣态按钮)= 收获 + 重派默认时长,**不空出小队**(要空出小队得等派遣到期自然完成)。

- 找不到副本/子副本:`choose_mission` 左右拖拽(`drag_to ±800`)翻页。

**开工态**(`制造坊.webp`):4 制造台(各带「开工」按钮 `414/756/1096/1438,y≈967`)+ 开物/售卖左右切换 + 自动收货中 + LEVEL Max。id_mark `标题-制造坊`(`1404,479`)/ `按钮-街区`(`238,26`,goto 大世界-普通)/ `菜单-返回`。

**选商品/邦布子态**(`制造坊-选商品.webp`,点「开工」后):⚠️ **标题「制造坊」移到左上 `178,27`** → 原 id_mark 位 `1404,479` 失效,此子态 `screens` 全误匹配。**设计如此**:`CraftDispatch.back_at_last` 靠「子态识别不到制造坊」判断「还没退出选商品页」,继续点确认/返回退回开工态(标题归位 → 命中 → success)。画面 = 左侧 Bangboo 邦布槽区(8 槽 `区域-邦布-1~8`)+ 商品列表(`区域-商品列表`,如 `[精淬]异构抗凝膏 100(可制造)`/`[精淬]镇厄面`/`[精淬]醒心促愈素`,含库存/可制造数)+ `请选择N号制造台的目标货品` + `开始制造` + 预计耗时。`区域-选择邦布`(`1220,237`)/ `按钮-制造后确认`(`1026,741`)在此子态。

**玩法(代码事实)**:
- `click_lets_go`:OCR 优先级 `['开工','开物','制造暂停']` **ignore `['开物']`** —— 开物是切 tab 不点,实际只点「开工」(可开工)或「制造暂停」(制造中)。

- 4 制造台逐一开工;无开工(都制造中)→ back。

**入口**:经营总览「制造」tab;**出口**:`菜单-返回` 回入口(或 `按钮-街区`→大世界)。

**玩法(代码事实)**:
- **价格上偏 -150**:`click_pos = 价格.center + Point(0,-150)` —— 点价格 OCR 中心上方 150px = 卡片本体(卡片在上、价格在卡底),非点价格本身。

- **drag 显鼠标**:购买动画的 `按钮-跳过`(`1680,105`)是逆天按钮(鼠标移上才显形可点),`drag_to(left_top→center)` 拖出鼠标再 `click`。
